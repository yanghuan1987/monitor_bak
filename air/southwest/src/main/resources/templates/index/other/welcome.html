简介: 以SpringBoot 为中心，模块化开发系统，用户可以随意删减除权限框架外 任意的系统模块。复用，组装性强。2种打包方式，传统可运行的tomcat目录 以及直接jar 方式运行。主要应用技术：SpringSecurity+Ehcache+quartz+swagger2+Mysql5.6.... 主要包含系统全方位监控，权限管理。定时任务，API在线接口，OSS云存储，第三方登录，在线支付,代码生成，工作流等内容。文档较全，适合单机型框架开发
模块化设计。兼容性强，支持多数据源。可扩展成spring cloud 分布式。

启动项目

默认端口8080

调试项目

右键 main 模块中的 App.java run as javaApplication 即可启动

打包成jar 启动

maven :单击quanmin_admin ,右键 maven build.., 执行 package 命令 打包成jar.详情见文档

变成tomcat 目录启动（不推荐）

切换成tomcat 部署的 pom ,maven :单击quanmin_admin ,右键 maven build.., 执行 package 命令 打包.打包之后在 main 模块的 /src/main/webapp 该webapp就是tomcat 目录，自行再次打包成war 或者使用tomcat server.xml 指向即可。maven 打包命令出现的war是不可用的

技术选型概要

为什么不用 Shiro? 本系统面向spring 系列，尽量少用 非 Spring 框架

为什么不用 Mybatis? 个人认为 Springjdbc 开发更方便

为什么使用Mysql?mysql 很小，很方便
